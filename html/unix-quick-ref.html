UNIX Quick Reference

Any term enclosed in angle brackets <like this> is a placeholder for some other value.

Logging in

Task Command
Securely connect to a computer over the network ssh <username>@<servername>
List users who are logged in w or who
View command history history
Close your connection logout

Navigating the filesystem

Task Command
Get your present working directory pwd
List files in your present working directory ls
List files at any other location ls <path>
View a long-listing of files ls -l <path>
List all files that start with a ls a*
List all files that end with .txt ls *.txt
Find all files that end with .txt below pwd find . *.txt
Change directory cd <path>
Move up one level cd ..
Move down one level cd <dirname>
Return to home directory cd or cd ~
Create a new directory mkdir <dirname>
Delete an empty directory rmdir <dirname>

Managing files

Task Command
Copy a file cp <source> <destination>
Copy a directory and all its contents cp -R <source> <dest>
Move or rename a file mv <source> <dest>
Delete a file (not reversible!) rm <file>
Copy a file between computers scp <user>@<server name>:<absolute path> <destination>
Start secure file transfer session sftp <user>@<servername>
List commands in sftp help
Stop sftp session exit
Show disk usage du
Show disk usage in a nicer format du -h
Show disk usage just one level down du -d1
Show disk usage at system level df

Running programs

The general anatomy of a command in UNIX is:
<program name> <arg1> <arg2> ... <argn>
where the program name and arguments are separated by spaces.
An argument can be an option with a - or -- prefix, a parameter, or a file path.

In the following table, I will occasionally use the cowsay program as an example. <Control> represents the "Control/Ctrl" key on your keyboard. <tab> represents the "tab" key.

Task Command
Read program documentation man <program name>
Exit man page q
Run a program cowsay moo
Run a program in the background <program> &
Autocomplete a program name cows<tab> moo
Display help text for a program cowsay -h or cowsay --help
Get the absolute path to the program file which cowsay
List directories checked for program files echo $PATH
Run a program in your current directory ./cowsay
Force exit a running program <Control>C
Release program to run in background <Control>Z
List running programs and their CPU usage top
Show free memory in a readable format free -h
List running programs and their process IDs ps
Stop a program running in background kill -9 <pid>
Redirect the program standard output (stdout) stream to another program <program 1> | <program 2>
Redirect stdout to a file (overwrites any previous content) <program> > <file>
Append stdout to a file <program> >> <file>

Inspecting files

Note, you probably don't want to run any of these commands on a binary executable (program) file.
The stdout (standard output) stream displays its contents to your console unless it is redirected to another program or file.

Task Command
Write the entire contents of a file to stdout cat <file>
Display the first few lines head <file>
Display exactly the first n lines head -n<number> <file>
Display the last few lines tail <fail>
Display exactly the last n lines tail -n<number> <file>
View contents one screen-full at a time more <file> (hit spacebar to advance, q to exit)
View contents in an interactive pager less <file>
Count the number of words, lines and characters wc <file>
Just show the number of lines wc -l <file>
Search for some pattern in a file grep <pattern> <file>
Search for pattern in any file at or below current directory grep -r <pattern> .
Count lines that match pattern grep -c <pattern> <file>
Compare two files diff <file1> <file2>
